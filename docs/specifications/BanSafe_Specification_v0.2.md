## 1. 项目概述

### 1.1 项目背景

BanSafe（伴星安全）是伴星成长教育系统的核心安全模块，专为 6-18 岁儿童及青少年设计。系统通过智能手表硬件、多传感器融合检测、室内定位技术和智能警报分发，全方位保护孩子在校内外的安全。

### 1.2 项目愿景

- **实时保护**：7×24 小时监测潜在危险
- **精准定位**：室内外无缝定位，误差±3-5 米
- **快速响应**：警报秒级推送至最近的 3-5 位责任人
- **隐私优先**：所有数据本地处理，零原始音视频上传
- **持久续航**：单电池续航≥72 小时（常规使用）

### 1.3 核心价值

| 价值维度 | 描述 |
|---------|------|
| 三重模态检测 | 姿态 + 麦克风 + 摄像头融合分析，准确识别危险场景 |
| 群体感知定位 | 利用周围手表的 BLE 信号实现校园内精确定位 |
| 智能分发 | 仅通知最相关的责任人，避免警报疲劳 |
| 隐私保护 | 所有敏感数据本地处理，仅上传结构化判定结果 |
| 低功耗设计 | 分层唤醒机制，确保 72 小时续航 |

### 1.4 技术标准

| 指标 | 目标值 |
|-----|-------|
| 误报率 | <0.5% |
| 定位精度 | ±5 米（BLE 模式）/ ±100 米（基站模式） |
| 响应时间 | <3 秒（从检测到警报推送） |
| 续航时间 | ≥72 小时（常规使用） |
| 隐私合规 | 《未成年人保护法》+ GDPR 儿童隐私条款 |

## 3. 三重判定引擎

### 3.1 概述

三重判定引擎是 BanSafe 的核心创新，通过姿态传感器、麦克风和摄像头的协同工作，确保仅在真实危险情况下触发警报，将误报率控制在 0.5% 以下。

### 3.3 姿态传感器检测

#### 3.3.1 传感器规格

| 参数 | 规格 |
|-----|------|
| 型号 | ICM-42688/P |
| 接口 | I2C (400kHz) |
| 加速度量程 | ±2g / ±4g / ±8g / ±16g |
| 陀螺仪量程 | ±125 / ±250 / ±500 / ±1000 / ±2000 dps |
| 工作电流 | 1.3mA |
| 休眠电流 | 1μA |

#### 3.3.2 检测模式

| 模式 | 特征 | 权重 |
|-----|------|------|
| **剧烈摇晃** | 加速度>2g，频率>2Hz | 80 分 |
| **跌倒检测** | 自由落体后突然静止 | 70 分 |
| **突然静止** | 运动状态突然停止>5 分钟 | 60 分 |
| **异常加速度** | 持续高加速度>3 秒 | 50 分 |
| **异常角度** | 设备角度持续异常>10 秒 | 40 分 |

#### 3.3.3 判定逻辑

```c
// 伪代码示例
typedef struct {
    float acceleration;    // 加速度值
    float gyro_x, gyro_y, gyro_z;  // 陀螺仪数据
    uint32_t duration;     // 持续时间 (ms)
} MotionEvent;

float calculate_posture_score(MotionEvent event) {
    float score = 0;
    
    // 剧烈摇晃检测
    if (event.acceleration > 2.0f && event.duration > 1000) {
        score += 80;
    }
    // 跌倒检测
    else if (is_free_fall() && is_sudden_stop()) {
        score += 70;
    }
    // 长时间静止
    else if (event.acceleration < 0.1f && event.duration > 300000) {
        score += 60;
    }
    // 异常加速度
    else if (event.acceleration > 1.5f && event.duration > 3000) {
        score += 50;
    }
    
    return min(score, 100);
}
```

#### 3.4.2 检测内容

| 检测项 | 阈值 | 权重 |

| **异常声音模式** | 尖叫声、撞击声 | 50 分 |
| **持续嘈杂** | 平均>70dB 持续>5 秒 | 40 分 |

#### 3.4.3 TinyML 关键词检测模型

| 参数 | 规格 |
|-----|------|
| 模型框架 | TensorFlow Lite Micro |
| 模型大小 | ~80KB |
| 支持关键词 | 救命、帮助、打人、救命啊、别打我、报警 |
| 推理时间 | <50ms |
| 准确率 | >90% |

#### 3.4.4 判定逻辑

```c
// 伪代码示例
typedef struct {
    float db_level;          // 分贝值
    float keyword_scores[6]; // 各关键词匹配度
    float anomaly_score;     // 异常声音分数
} AudioAnalysisResult;

float calculate_audio_score(AudioAnalysisResult result) {
    float score = 0;
    
    // 高分贝检测
    if (result.db_level > 80.0f) {
        score += 60;
    }
    // 关键词匹配
    for (int i = 0; i < 6; i++) {
        if (result.keyword_scores[i] > 0.8f) {
            score += 70;
            break;
        }
    }
    // 异常声音
    if (result.anomaly_score > 0.7f) {
        score += 50;
    }
    
    return min(score, 100);
}
```

```c
// 伪代码示例
typedef struct {
    float conflict_score;    // 冲突概率
    float crowd_score;       // 聚集概率
    float pose_score;        // 异常姿势概率
    float object_score;      // 危险物品概率
} VisionAnalysisResult;

float calculate_vision_score(VisionAnalysisResult result) {
    float score = 0;
    
    // 肢体冲突
    if (result.conflict_score > 0.7f) {
        score += 80;
    }
    // 多人聚集
    if (result.crowd_score > 0.6f) {
        score += 60;
    }
    // 异常姿势
    if (result.pose_score > 0.6f) {
        score += 50;
    }
    // 危险物品
    if (result.object_score > 0.7f) {
        score += 70;
    }
    
    return min(score, 100);
}
```

### 3.6 多模态融合算法

#### 3.6.1 加权融合公式

```
综合评分 = 姿态评分 × 30% + 麦克风评分 × 40% + 摄像头评分 × 30%
```

#### 3.6.2 判定决策表

| 姿态 | 麦克风 | 摄像头 | 综合评分 | 判定结果 |
|-----|-------|-------|---------|---------|
| ≥40 | - | - | - | 进入麦克风检测 |
| ≥40 | ≥20 | - | ≥60 | 进入摄像头检测 |
| ≥40 | ≥35 | - | ≥75 | **直接触发警报**（无摄像头模式） |
| ≥40 | ≥20 | ≥20 | ≥80 | **触发警报** |
| <40 | - | - | - | 忽略 |
| ≥40 | <20 | - | <60 | 忽略 |
| ≥40 | ≥20 | <20 | <80 | 忽略 |

#### 3.6.3 融合判定伪代码

```c
typedef enum {
    PHASE_POSTURE = 0,
    PHASE_AUDIO,
    PHASE_VISION,
    PHASE_TRIGGER
} DetectionPhase;

typedef struct {
    float posture_score;
    float audio_score;
    float vision_score;
    float final_score;
} DangerAssessment;

DetectionPhase process_detection(DangerAssessment *assessment) {
    // 阶段 1：姿态检测
    assessment->posture_score = calculate_posture_score(get_motion_event());
    if (assessment->posture_score < 40) {
        return PHASE_POSTURE; // 忽略
    }
    
    // 阶段 2：麦克风检测
    assessment->audio_score = calculate_audio_score(analyze_audio(10));
    
    // 无摄像头模式：姿态 + 麦克风≥75 直接触发
    float two_phase_score = assessment->posture_score * 0.5f + 
                            assessment->audio_score * 0.5f;
    if (two_phase_score >= 75) {
        assessment->final_score = two_phase_score;
        return PHASE_TRIGGER;
    }
    
    // 进入阶段 3：摄像头检测
    if (assessment->posture_score * 0.5f + assessment->audio_score * 0.5f >= 60) {
        close_microphone();  // 释放 I2S 资源
        open_camera();       // 启用 DCMI
        assessment->vision_score = calculate_vision_score(analyze_video(15));
        close_camera();
        
        // 三模态融合
        assessment->final_score = assessment->posture_score * 0.3f +
                                  assessment->audio_score * 0.4f +
                                  assessment->vision_score * 0.3f;
        
        if (assessment->final_score >= 80) {
            return PHASE_TRIGGER;
        }
    }
    
    return PHASE_VISION; // 未触发
}
```

#### 4.3.2 RSSI-距离模型

```c
// RSSI 转距离公式
float rssi_to_distance(int8_t rssi, int8_t rssi_ref) {
    // 自由空间路径损耗模型
    float ratio = (float)(rssi_ref - rssi) / 20.0f;
    return powf(10.0f, ratio);
}

// 滑动平均滤波（减少 RSSI 波动）
int8_t filter_rssi(int8_t *history, int count) {
    float sum = 0;
    for (int i = 0; i < count; i++) {
        sum += history[i];
    }
    return (int8_t)(sum / count);
}
```

### 4.4 楼层判定（气压计辅助）

```c
// 气压计楼层判定
typedef struct {
    float sea_level_pressure;  // 海平面气压（校准值）
    float current_pressure;    // 当前气压
} BarometerData;

int calculate_floor(BarometerData data) {
    // 气压差转高度差（每层楼约 3 米）
    float height_diff = 44330.0f * (1.0f - 
                        powf(data.current_pressure / data.sea_level_pressure, 0.1903f));
    
    // 假设一层楼约 3 米，地面层为 0
    int floor = (int)(height_diff / 3.0f) + 1;
    return max(floor, 1);
}
```

## 附录 A：数据隐私保护

### A.1 数据最小化原则

- 仅收集必要的安全数据
- 位置数据仅在警报触发时记录
- 声音数据不存储，仅用于实时分析
- 所有原始音视频数据本地处理，不上传云端

### A.2 数据保留政策

| 数据类型 | 保留期限 | 说明 |
|---------|---------|------|
| 警报记录 | 1 年 | 用于事故追溯 |
| 位置数据 | 24 小时 | 仅保留最近 24 小时 |
| 传感器原始数据 | 不存储 | 仅用于实时分析 |
| 用户信息 | 永久 | 用户主动删除前保留 |

### A.3 数据安全

- 传输加密：TLS 1.3
- 存储加密：AES-256
- 设备认证：双向证书认证
